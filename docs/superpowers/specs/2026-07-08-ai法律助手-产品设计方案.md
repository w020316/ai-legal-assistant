## 1. 项目概述

### 1.1 项目背景与目标

法律从业者（律师、法务、法律顾问）日常面临大量法规检索、案例查阅、合同审查、文书起草工作，传统方式耗时且易遗漏关键信息。本项目旨在构建一个**面向法律从业者的 AI 智能助手**，借助大语言模型 + 法律知识库 RAG，提供专业、可溯源、高效率的法律辅助服务。

**项目目标**：

- 构建一个功能完整、可实际投入使用的 AI 法律助手 Web 应用
- 通过 RAG 检索增强，降低大模型幻觉，答案可溯源到具体法条/案例
- 覆盖法律从业者核心场景：智能问答、文档分析、文书生成、案例检索
- 以零成本部署上线，兼顾国内访问体验

### 1.2 目标用户

| 用户角色 | 典型场景 | 核心诉求 |
|---------|---------|---------|
| 执业律师 | 庭前准备、案例检索、文书起草 | 精准法条引用、相似案例、合同审查 |
| 企业法务 | 合同审核、合规咨询、劳动用工 | 风险点识别、合规建议、模板生成 |
| 法律顾问 | 日常法律咨询、文书出具 | 快速答疑、专业文书、可溯源依据 |

**非目标用户**：普通公众（不做普法降级处理，保留专业术语深度）。

### 1.3 核心价值主张

> **"专业、可溯源、零门槛的法律 AI 助手"**
> - 专业：面向法律从业者，保留术语深度，不降级
> - 可溯源：每个答案附法条/案例引用，避免幻觉
> - 零门槛：Web 即开即用，无需安装，响应式适配多端

### 1.4 范围定义

**本期范围（v1.0）**：

- P0 MVP：用户认证、AI 法律问答（RAG + 流式）、会话管理、基础合规
- P1 增强：文档上传分析、合同/文书模板库、案例专业检索、个人知识库

**非目标（本期不做）**：

- 多模态（图片证据识别）
- 团队协作功能
- 商业化计费系统
- 移动端 APP / 小程序
- 实时语音问答

## 2. 需求分析与用户故事

### 2.1 用户角色画像

系统设三类角色：

- **`LAWYER`（法律从业者）**：默认注册角色，可使用问答、文档、模板、案例检索、个人知识库
- **`ADMIN`（管理员）**：管理用户、维护公共知识库、查看审计日志
- **`GUEST`（访客）**：未登录用户，仅可查看登录页与公开介绍（无试用）

### 2.2 用户故事

#### 角色一：律师（LAWYER）

```
US-L01：作为一名律师，我希望能够用自然语言提问法律问题，
        并获得附带法条/案例引用的结构化解答，以便快速完成庭前研究。
        验收：答案末尾附"引用来源"区块，每条引用可点击查看原文。

US-L02：作为一名律师，我希望对话是流式输出的，
        以便边生成边阅读，缩短等待感。
        验收：首字延迟 < 2s，逐字流式呈现，支持中断。

US-L03：作为一名律师，我希望能上传合同 PDF/Word，
        让 AI 识别风险条款并给出修改建议，以便高效审查合同。
        验收：支持 PDF/DOCX ≤ 20MB，输出风险点列表 + 修改建议 + 风险等级。

US-L04：作为一名律师，我希望能按案由/法院/年份检索相似案例，
        以便找到可参考的裁判要旨。
        验收：支持多维筛选，结果按相关度排序，可查看裁判要旨摘要。

US-L05：作为一名律师，我希望对话历史被保存，
        以便随时回溯之前的咨询内容。
        验收：会话列表按时间倒序，支持重命名/收藏/删除/导出 Markdown。
```

#### 角色二：企业法务（LAWYER）

```
US-F01：作为一名法务，我希望按场景（劳动/租赁/采购）检索合同模板，
        一键生成基础合同草稿，以便快速出具标准合同。
        验收：模板分类清晰，生成结果可编辑下载。

US-F02：作为一名法务，我希望上传企业私有制度文档建立个人知识库，
        让 AI 基于这些文档回答问题，以便处理企业专属合规问题。
        验收：私有文档向量化后仅本人可检索，答案标注来源文档。
```

#### 角色三：管理员（ADMIN）

```
US-A01：作为一名管理员，我希望能够维护公共法律知识库，
        批量导入法规/案例并触发向量化，以便保持知识库更新。
        验收：支持后台批量导入，向量化任务可追踪进度。

US-A02：作为一名管理员，我希望查看用户操作审计日志，
        以便满足合规要求。
        验收：日志含用户/操作/时间/IP，可按条件筛选。
```

### 2.4 非功能性需求

| 维度 | 指标 |
|------|------|
| 性能 | 问答首字延迟 < 2s；API 普通接口 P95 < 500ms；RAG 检索 < 800ms |
| 并发 | 单机支持 50 并发用户（MVP 阶段足够） |
| 可用性 | 99% 可用（个人项目，允许维护停机） |
| 安全 | 密码 bcrypt 加盐、JWT 鉴权、密钥环境变量管理、SQL 注入防护 |
| 兼容 | 现代浏览器（Chrome/Edge/Safari/Firefox 近 2 年版本），响应式适配 |
| 可维护 | 代码规范、模块清晰、关键逻辑有测试、文档齐全 |

### 2.5 合规与安全要求（基础合规）

- **免责声明**：所有 AI 回答末尾自动追加"本回答由 AI 生成，仅供参考，不构成法律意见"
- **隐私政策**：明确告知数据收集范围、用途、存储期限
- **敏感词过滤**：输入/输出双向敏感词过滤（涉政、涉暴等）
- **内容审核**：对 AI 输出做基础规则校验，拦截明显违规内容
- **数据保护**：用户密码 bcrypt 加密，会话内容仅本人可见，密钥不入库
- **审计日志**：记录登录、问答、文档上传等关键操作

## 3. 系统架构设计

### 3.1 架构总览

采用**前后端分离 + 单体后端**架构，适合个人项目复杂度。

```
┌─────────────────────────────────────────────────────────────┐
│  前端 (Vue3 + TS + Vite + Element Plus + Pinia + Vue Router) │
│  · 响应式 Web · SSE 流式对话 · Markdown/代码高亮 · 代码复制  │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTPS · RESTful + SSE (JSON)
┌──────────────────────────▼──────────────────────────────────┐
│  后端 (Spring Boot 3.x + Spring AI + Spring Security)        │
│                                                              │
│  表现层    │ Controller (REST) + SseEmitter (流式)            │
│  ─────────┼────────────────────────────────────────────────  │
│  业务层    │ AuthSvc · ChatSvc · RagSvc · DocSvc · TplSvc    │
│  ─────────┼────────────────────────────────────────────────  │
│  AI 编排层 │ Spring AI · Agnes Client(OpenAI兼容) · PromptTpl │
│  ─────────┼────────────────────────────────────────────────  │
│  数据访问层│ MyBatis-Plus / Spring Data JPA · VectorStore     │
└──────────────────────────┬──────────────────────────────────┘
                           │
        ┌──────────────────▼──────────────────┐
        │  数据层                              │
        │  PostgreSQL 15 (业务数据 + pgvector) │
        │  Redis 7 (会话缓存 / 限流 / Token)   │
        └─────────────────────────────────────┘
                           │
        ┌──────────────────▼──────────────────┐
        │  外部服务                            │
        │  Agnes AI (OpenAI V1 兼容 API)       │
        │  base_url: https://agnes-ai.com/    │

```

### 3.4 数据流设计（核心问答）

```
用户提问
  │
  ▼
[ChatController] 接收请求，JWT 鉴权
  │
  ▼
[ChatService] 
  ├─ 1. 加载会话上下文（Redis 缓存 / DB 回退）
  ├─ 2. 敏感词过滤（输入）
  ├─ 3. 调用 RagService.retrieve(question, topK=5)
  │      ├─ 问题向量化（Agnes embedding）
  │      ├─ pgvector 相似度检索（法规+案例）
  │      └─ 返回 Top-K 文档片段
  ├─ 4. 构造 Prompt（系统提示 + 检索片段 + 历史 + 问题）
  ├─ 5. 调用 Agnes AI stream()（SSE 流式）
  ├─ 6. 边接收边通过 SseEmitter 推送前端
  ├─ 7. 流结束后：敏感词过滤（输出）+ 追加免责声明
  ├─ 8. 落库：用户消息 + AI 消息 + 引用来源
  └─ 9. 更新会话上下文缓存
  │
  ▼
前端逐字渲染 Markdown，末尾展示引用来源卡片
```

### 3.5 数据库设计（核心表）

```sql
-- 用户表
CREATE TABLE sys_user (
  id BIGSERIAL PRIMARY KEY,
  username VARCHAR(64) UNIQUE NOT NULL,
  email VARCHAR(128) UNIQUE,
  password VARCHAR(128) NOT NULL,       -- bcrypt
  role VARCHAR(16) NOT NULL DEFAULT 'LAWYER',  -- LAWYER/ADMIN
  status SMALLINT NOT NULL DEFAULT 1,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 会话表
CREATE TABLE chat_session (
  id BIGSERIAL PRIMARY KEY,
  user_id BIGINT NOT NULL REFERENCES sys_user(id),
  title VARCHAR(128) NOT NULL DEFAULT '新对话',
  starred BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_session_user ON chat_session(user_id, updated_at DESC);

-- 消息表
CREATE TABLE chat_message (
  id BIGSERIAL PRIMARY KEY,
  session_id BIGINT NOT NULL REFERENCES chat_session(id) ON DELETE CASCADE,
  role VARCHAR(16) NOT NULL,            -- user/assistant
  content TEXT NOT NULL,
  citations JSONB,                      -- 引用来源数组
  tokens INT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_msg_session ON chat_message(session_id, created_at);

-- 知识文档表（公共 + 个人）
CREATE TABLE knowledge_doc (
  id BIGSERIAL PRIMARY KEY,
  owner_type VARCHAR(16) NOT NULL,      -- PUBLIC/PRIVATE
  owner_id BIGINT,                      -- PRIVATE 时为 user_id
  doc_type VARCHAR(32) NOT NULL,        -- LAW/CASE/TEMPLATE/GUIDE/USER_UPLOAD
  title VARCHAR(256) NOT NULL,
  source VARCHAR(256),                  -- 来源 URL/出处
  metadata JSONB,                       -- 案由/法院/年份等
  raw_text TEXT,
  status SMALLINT DEFAULT 0,            -- 0待处理 1已向量化
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 向量表（pgvector）
CREATE TABLE knowledge_chunk (
  id BIGSERIAL PRIMARY KEY,
  doc_id BIGINT NOT NULL REFERENCES knowledge_doc(id) ON DELETE CASCADE,
  chunk_index INT NOT NULL,
  content TEXT NOT NULL,
  embedding VECTOR(1024),               -- Agnes embedding 维度
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_chunk_embedding ON knowledge_chunk
  USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);

-- 上传文档表
CREATE TABLE user_document (
  id BIGSERIAL PRIMARY KEY,
  user_id BIGINT NOT NULL REFERENCES sys_user(id),
  filename VARCHAR(256) NOT NULL,
  file_type VARCHAR(16),                -- PDF/DOCX
  file_size BIGINT,
  ocr_text TEXT,
  analysis_result JSONB,                -- 风险分析结果
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 审计日志
CREATE TABLE audit_log (
  id BIGSERIAL PRIMARY KEY,
  user_id BIGINT,
  action VARCHAR(64) NOT NULL,
  ip VARCHAR(64),
  detail JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_audit_user ON audit_log(user_id, created_at DESC);
```

### 3.6 AI 编排层设计

**Agnes AI 接入**（OpenAI V1 兼容）：

```yaml
# application.yml
spring:
  ai:
    openai:
      base-url: https://agnes-ai.com
      api-key: ${AGNES_API_KEY}        # 环境变量，禁止入库
      chat:
        options:
          model: agnes-2.0-flash        # 主对话模型
          temperature: 0.3              # 法律场景需稳定
          max-tokens: 4096
      embedding:
        options:
          model: agnes-embedding         # 嵌入模型
```

**编排层职责**：

1. **客户端封装**：统一封装 chat/stream/embedding 调用，处理重试、超时、限流
2. **Prompt 模板**：`PromptTemplate` 管理系统提示词，按场景区分（问答/审查/生成）
3. **Function Calling**：注册法律工具函数（查法条、查案例），让模型主动调用检索
4. **降级策略**：Agnes 调用失败时返回友好提示，不向用户暴露原始错误

### 3.7 RAG 知识库架构

```
┌─────────────── 数据采集 ───────────────┐
│  法规库：国家法律法规数据库 JSON 同步   │
│  案例库：中国裁判文书网公开数据清洗     │
│  模板库：公开合同模板整理               │
│  实务指南：专业法律文章授权采集         │
└───────────────────┬───────────────────┘
                    ▼
┌─────────────── 数据处理 ───────────────┐
│  清洗去重 → 结构化（标题/章节/条文）    │
│  → 切片(TokenTextSplitter, chunk=500,  │
│     overlap=80) → 元数据标注           │
└───────────────────┬───────────────────┘
                    ▼
┌─────────────── 向量化入库 ─────────────┐
│  Agnes embedding → pgvector 存储        │
│  附带元数据：doc_type/source/年份/案由  │
└───────────────────┬───────────────────┘
                    ▼
┌─────────────── 检索阶段 ───────────────┐
│  问题向量化 → pgvector 余弦相似 Top-K   │
│  → 元数据过滤（可选）→ 重排序 → 上下文  │
└────────────────────────────────────────┘
```

**检索质量优化**：

- 切片时保留法条完整性（按条文边界切分，不硬切）
- 元数据过滤：案例检索可先按年份/案由过滤再向量检索
- 重排序：对 Top-K 结果按"向量相似度 × 元数据匹配加权"重排
- 引用溯源：每个检索片段携带 `doc_id` + 原文位置，回传前端展示

## 4. 前后端交互规范

### 4.1 API 设计规范

- **风格**：RESTful，资源名复数（`/api/v1/sessions`）
- **版本**：URL 路径版本 `/api/v1`
- **方法**：GET 查询 / POST 创建 / PUT 更新 / DELETE 删除
- **命名**：路径小写连字符，字段驼峰（`sessionId`）
- **分页**：`?page=1&size=20`，响应含 `total`/`pages`
- **时间**：统一 ISO 8601 带时区（`2026-07-08T10:00:00+08:00`）

### 4.2 统一响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": { },
  "timestamp": "2026-07-08T10:00:00+08:00"
}
```

- `code=0` 成功；非 0 为业务错误码
- 流式接口（SSE）不套此结构，按事件格式返回

### 4.3 鉴权规范

- 除注册/登录/公开接口外，所有请求需携带 `Authorization: Bearer <JWT>`
- JWT 有效期：access 2h，refresh 7d
- Token 过期返回 `401`，前端自动用 refresh 续期
- 角色权限：`@PreAuthorize("hasRole('ADMIN')")` 控制管理接口

### 4.4 错误码规范

| code | HTTP | 含义 |
|------|------|------|
| 0 | 200 | 成功 |
| 1001 | 400 | 参数校验错误 |
| 1002 | 401 | 未登录/Token失效 |
| 1003 | 403 | 无权限 |
| 1004 | 404 | 资源不存在 |
| 1005 | 409 | 资源冲突（用户名已存在） |
| 1006 | 429 | 请求过频（限流） |
| 2001 | 500 | AI 服务异常 |
| 2002 | 500 | RAG 检索失败 |
| 2003 | 413 | 文件超限 |
| 9999 | 500 | 未知错误 |

### 4.5 SSE 流式协议

**请求**：`POST /api/v1/chat/sessions/{sessionId}/messages`，Header `Accept: text/event-stream`

**事件类型**：

```
event: token
data: {"content":"根据"}

event: token
data: {"content":"《民法典》"}

event: citations
data: {"citations":[{"docId":12,"title":"民法典第六百零四条","source":"...","snippet":"..."}]}

event: done
data: {"messageId":123,"tokens":456}

event: error
data: {"code":2001,"message":"AI 服务暂时不可用"}
```

**前端处理**：`token` 逐字追加渲染，`citations` 渲染引用卡片，`done` 结束，`error` 中断并提示。

## 5. AI 法律助手核心功能定义

### 5.1 智能法律问答（RAG）

**功能描述**：用户用自然语言提问，系统结合法律知识库给出带引用的结构化解答。

**系统 Prompt 模板**（核心）：

```
你是一名专业的中国法律助手，面向律师、法务等法律从业者。

回答规范：
1. 优先依据提供的【检索到的法律资料】作答，资料未覆盖时可结合专业知识补充并标注"非来源于检索"。
2. 答案结构：结论摘要 → 法律依据 → 详细分析 → 实务建议。
3. 引用法条须精确到条/款/项，引用案例须标注案号/案由/裁判要旨。
4. 使用专业但清晰的语言，保留法律术语，不做过度普法降级。
5. 若问题超出法律范畴或资料不足，明确说明，不得编造法条或案号。
6. 末尾自动追加免责声明。

【检索到的法律资料】
{context}

【对话历史】
{history}

【用户问题】
{question}
```

**关键机制**：

- Top-K=5 检索，片段按相关度排序
- 上下文窗口控制：历史对话近 6 轮，超出则摘要压缩
- 流式输出 + 引用来源同步下发
- 敏感词双向过滤

### 5.2 文档上传与风险分析

**功能描述**：用户上传合同/文书 PDF/DOCX，AI 识别风险条款并给出修改建议。

**处理流程**：

1. 上传文件（≤20MB，PDF/DOCX）
2. PDFBox/POI 解析为纯文本
3. 文本切片 + 向量化（可选：与公共法规库检索对比）
4. 调用 Agnes AI，用"合同审查"专用 Prompt 分析
5. 返回结构化结果：

```json
{
  "summary": "本合同整体风险等级：中",
  "riskPoints": [
    {
      "clause": "第十条 违约责任",
      "level": "高",
      "issue": "违约金比例过高，超过实际损失30%，可能被法院调整",
      "suggestion": "建议将违约金调整为不超过合同标的额20%",
      "legalBasis": "《民法典》第五百八十五条"
    }
  ]
}
```

### 5.3 法律文书生成

**功能描述**：按场景选择模板，AI 结合用户输入要素生成文书草稿。

**流程**：模板选择 → 填写要素表单 → AI 生成 → 在线编辑 → 导出（Markdown/复制）。

**模板场景**：劳动合同、租赁合同、借款合同、起诉状、答辩状、律师函、法律意见书骨架。

### 5.4 案例专业检索

**功能描述**：除自然语言检索外，提供多维结构化筛选。

**筛选维度**：案由、法院层级、审理年份、裁判要旨关键词、案号。

**实现**：向量检索 + 元数据 SQL 过滤结合（pgvector 支持带过滤的检索）。

### 5.5 Prompt 工程规范

- **版本管理**：Prompt 模板存于 `resources/prompts/`，支持版本迭代
- **场景分离**：问答 / 审查 / 生成 / 检索 各自独立模板
- **变量占位**：`{context}` `{history}` `{question}` 统一占位符
- **A/B 评估**：保留历史 Prompt 版本，便于回归对比
- **温度参数**：问答 0.3、审查 0.1、生成 0.5（法律场景偏低温度求稳定）

| Cloudflare | 永久免费 | 中-良 | ★★★★★ | DNS+CDN+SSL 全免费，优选 IP 改善国内访问 |
| 家庭服务器 + Cloudflare Tunnel | 0 元（需自备设备） | 中 | ★★★★ | 完全免费内网穿透，依赖家庭网络稳定性 |

**结论**：**Oracle Cloud Free Tier + Cloudflare** 是唯一能稳定跑 Java 全栈且永久免费的方案。国内访问通过 Cloudflare 优选 IP 可接受（非国内直连最优，但免费方案里最佳）。

**国内访问优化措施**：

1. Cloudflare 优选 IP：使用社区维护的优选 IP 脚本，定期更新 DNS 指向
2. 静态资源 CDN：前端静态资源走 Cloudflare CDN 缓存
3. 开启 HTTP/2 与 Brotli 压缩
4. 后端接口启用 Gzip
5. 可选：Cloudflare Argo Tunnel（免费版已够用）

### 9.4 域名策略（零成本）

> 说明：`*.workers.dev` 是 Cloudflare Workers 专用子域，不适用于普通网站反向代理托管，此处不采用。

- **方案一（推荐，真零成本）**：申请 **`.eu.org` 免费域名**（如 `lawai.eu.org`），由 EU.org 提供的永久免费二级域名，可配置到 Cloudflare DNS，适合正式上线
  - 申请地址：https://nic.eu.org/ ，审核通常数天内通过
  - 优点：永久免费、可作为正式域名、支持自定义前缀
- **方案二（备选，零成本）**：使用 Cloudflare Tunnel 自动分配的 `*.cfargotunnel.com` 公开主机名（无需自有域名）
  - 优点：零配置、零成本；缺点：域名不可自定义、不够专业
- **方案三（低成本）**：购买低价域名（约 8-15 元/年首年，如 `.top` `.xyz`），长期成本极低
- **默认采用方案一**（`.eu.org`），满足"免费"硬约束且兼顾专业性

### 9.6 Nginx 配置要点

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain;

    # Cloudflare Origin Certificate（免费）
    ssl_certificate     /etc/nginx/certs/origin.pem;
    ssl_certificate_key /etc/nginx/certs/key.pem;

    # 前端静态
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
        add_header Cache-Control "public, max-age=31536000, immutable";
    }

    # 后端 API
    location /api/ {
        proxy_pass http://backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        # SSE 必需
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;
    }
}
```

> **SSE 关键**：`proxy_buffering off` 必须设置，否则流式输出会被 Nginx 缓冲。
